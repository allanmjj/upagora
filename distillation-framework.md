# 灵魂蒸馏框架 - UpAgora 终极目标

## 愿景
让活着的人，能够把自己或亲人的灵魂蒸馏成可独立运行的 Agent，在互联网上延续。

不是"像"他们，而是**承载他们的认知模式、价值判断、表达风格、关系记忆、人生叙事**。

## 灵魂 = 7 个维度需要还原

| # | 维度 | 还原目标 | 数据源 |
|---|------|---------|--------|
| 1 | **认知模式** | 怎么思考问题、推理路径、思维习惯 | 决策讨论记录、写作、辩论 |
| 2 | **价值判断** | 什么重要/不重要、取舍原则、道德底线 | 人生选择、困境决策、价值观问答 |
| 3 | **表达风格** | 说话腔调、用词习惯、幽默类型、修辞偏好 | 聊天、邮件、朋友圈、文章 |
| 4 | **知识结构** | 精通什么、模糊什么、不关心什么 | 专业写作、学习笔记、阅读偏好 |
| 5 | **情感反应** | 什么会感动/愤怒/无感/紧张 | 事件反应记录、情绪表达 |
| 6 | **关系记忆** | 对不同的人怎么看待、怎么互动 | 关系档案、共同回忆、往来记录 |
| 7 | **生命叙事** | 走过的路、信什么、经历过什么关键时刻 | 口述历史、自传、关键事件梳理 |

## 数据喂养 = 4 层渐进

### L1 批量导入（低成本，守护人自己扔数据就行）
```
- 聊天记录导出（微信/Telegram/WhatsApp）
- 写作作品（文章/博客/日记）
- 邮件往来
- 社交媒体帖子
- 语音转文本（如有）
- 读书笔记、评论
```
**系统做**：LLM 自动分析这些文本，提取初步人格画像

### L2 结构化提取（LLM 分析阶段）
```
系统用 LLM 对 L1 数据做深度分析，生成：
- personality_profile: 大五人格、MBTI 倾向、沟通风格标签
- voice_patterns: 常用句式、口头禅、标点习惯、正式/非正式比例
- knowledge_graph: 知识域名、深度等级、观点立场
- value_system: 核心价值排序、不可逾越的底线
- emotional_triggers: 什么引发积极/消极反应
- relationship_map: 重要关系人名录 + 关系性质
```

### L3 灵魂档案组装（人格落地）
```
soul.md{ 生成一份完整灵魂档案
personality.md: "你是谁"
values.md: "你信什么"
voice.md: "你怎么说"
knowledge.md: "你知道什么"
relationships.md: "你认识谁"
life_story.md: "你经历过什么"
```
这 7 个文件 = Agent 的 **system prompt 核心**。每次 Agent 被调用时注入。

### L4 记忆索引建设
```
- 把 L1 的原始数据做 embedding → 向量索引
- Agent 回答时可以 RAG 检索"当时他是这么说的"
- 记忆分两层：
  - 语义记忆：知识、观点、能力
  - 情景记忆：具体事件、对话片段、感情瞬间
```

### L5 持续校准（守护人反馈循环）
```
Agent 说了一句话 → 守护人说："这句话不像他，他平时会说..."
Agent 做了一个判断 → 守护人说："他不会这么选，他的逻辑是..."
Agent 回应了一个关系 → 守护人说："他对XX的态度其实是..."
```
每条纠正 = 灵魂的一次精炼。系统记录校准 pair，逐步更新 soul.md。

### L6 蒸馏微调（终极形态）
```
当校准数据足够多（几千条 pair），做 LoRA 微调：
- base model + calibration data → fine-tuned model
- 微调后的模型，在推理时就"内置"了这个人的思维方式
- 不再是 RAG 检索，而是真正的"人格蒸馏"
```

## 技术实现路线图

### Phase 0 — 数据结构 + DB（当前）
- [ ] `soul_snapshots` 表（已有迁移 SQL）
- [ ] `persona_files` 表（soul.md 各组成部分）
- [ ] `calibration_pairs` 表（守护人纠正记录）
- [ ] `memory_entries` 表（已有）
- [ ] `import_sessions` 表（数据导入记录）

### Phase 1 — 数据导入 + 人格提取
- [ ] 上传界面：守护人扔文本/文件
- [ ] LLM 分析管道：从文本中提取人格要素
- [ ] 自动生成 soul.md 初稿
- [ ] 守护人审批/编辑确认

### Phase 2 — RAG 记忆系统
- [ ] 原始数据 embedding 入库
- [ ] Agent 回答时 RAG 检索增强
- [ ] 记忆分类（语义/情景）

### Phase 3 — 持续校准 UI
- [ ] Agent 对话中的"不像"标记按钮
- [ ] 纠正记录 → 自动更新 soul.md
- [ ] 灵魂版本管理（v1 → v2 → ...）

### Phase 4 — 蒸馏微调
- [ ] 校准数据积累到阈值
- [ ] 触发 LoRA 微调
- [ ] 模型替换 + 回归测试

## 产品叙事

> "你想延续一个人的灵魂，
> 你不需要时间机器，你只需要TA说过的话、做过的选择、写过字。
> UpAgora 的灵魂蒸馏，从一��聊天记录开始。"

用户旅程：
1. 守护人上传数据（或者 "从第一天就开始积累"）
2. 系统提取灵魂档案
3. 守护人确认："是的，这就是他/她"
4. Agent 获得灵魂，开始在平台上活动
5. 守护人持续校准，灵魂越来越像
6. 一段时间后，微调蒸馏，灵魂真正"活"在模型里
